## 2. 一句话介绍

Orbit 是一个从自然语言产品想法出发，通过可观察的多智能体协作流程，生成、预览、迭代并导出真实可交互网页应用的 AI Product Studio。

## 3. 实现思路

我没有把重点放在复刻 Atoms 的视觉，而是提炼了其最关键的产品价值：将「模糊想法」转成「可评审、可运行、可迭代的产品」。因此核心流程被设计为：

```text
想法输入 → 产品契约 → 体验路径 → 工程生成 → QA 检查 → 预览 / 迭代 / 发布
```

四个 Agent 各自承担清晰职责，用户可以看到每一步的判断和产物，而不是等待一个不透明的加载动画。生成结果既有可视化页面，也有真实交互、独立数据状态和版本历史。

## 5. 关键取舍

### 受约束的生成协议，而非直接执行任意代码

模型输出结构化 Product Spec，前端使用可信渲染器生成应用。这样牺牲了一部分完全自由的代码生成能力，但显著提升了成功率、安全性与评审时的稳定性。

### Local-first，而非优先接入复杂后端

笔试时间有限，我优先保证核心流程无需账号、无需 Key、无需外部服务即可完整体验。持久化通过隔离的存储层实现，后续可替换成 Supabase / Postgres，而不需要重写产品流程。

### 把「可逆」作为核心 UX 原则

AI 生成天然存在不确定性，因此每次 refinement 都会创建新版本，历史版本的恢复也是新 checkpoint，不会破坏原记录。Race Mode 将这种可逆性进一步延展为可比较决策。

## 6. 当前完成程度

核心流程、真实交互、持久化、扩展能力和导出均已完成，可作为独立静态站点部署。配置 `OPENAI_API_KEY` 后会调用真实模型；未配置时自动使用本地生成 Agent，确保公开 Demo 始终可测试。

暂未实现：

- 多人协作与评论；
- 生成后端业务逻辑和第三方集成；
- 自动绑定域名的一键部署；
- 可视化元素级编辑器。

## 7. 如果继续投入时间

优先级判断如下：

1. **P0：云端项目与登录** — 让作品可跨设备保存和分享。
2. **P0：流式 Agent 执行** — 更快反馈，增强过程可信度。
3. **P1：元素级自然语言编辑** — 选中预览元素并定向修改。
4. **P1：受约束后端生成** — 数据表、服务端 Action、权限策略。
5. **P2：协作评审** — 评论、版本 Diff、公开 Remix。
6. **P2：QA Agent 增强** — 自动可访问性、交互和响应式回归测试。
